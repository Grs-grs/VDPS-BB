# Findings

Use this directory to separate confirmed issues from observations.

## Files

- `findings_draft.md`: active working narrative and request accounting.
- `confirmed.md`: confirmed vulnerabilities ready for report drafting.
- `suspected.md`: plausible issues that need safe validation or human review.
- `observations.md`: non-vulnerability context worth preserving.
- `false_positives.md`: checked items that should not be reported.

Evidence must be minimal and redacted. Do not store secrets, session cookies, personal data, customer data, or large raw response bodies here.
